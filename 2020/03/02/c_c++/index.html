    一. 预处理

    1. 文件包含

    预处理指令#include用于包含头文件，有两种形式：#include <xxx.h>，#include “xxx.h”。 

    #include <xxx.h>                  //标准或系统提供的头文件
#include "xxx.h"                  //用户自己创造的头文件

    后者“ ”中可以指出文件路径和文件名。如果在双引号中没有给出绝对路径，则默认为用户当前目录中的文件，此时系统首先在用户当前目录中寻找要包含的文件，若找不到再在系统目录中查找。

    2. 条件编译

    #if  表达式                             //表达式值为1便编译下面代码
#ifdef xxx                             //xxx已被宏定义就编译下面代码
#ifndef xxx                            //xxx未被宏定义就编译下面代码
#else                                  //作为其它预处理的剩余选项进行编译
#elif  表达式                           //这是一种#else和#if的组合选项只能与#if连用
#endif                                  //结束编译块的控制

    eg：

    //you.h
#include <stdio.h>
void is_tool()
{
    printf("hello world");
}


//guang.h
#include "you.h"


//main.cpp
#include "you.h"
#include "guang.h"
int main()
{
    is_tool();
    return 0;
}

    将上述代码进行编译链接后就相当于下面的代码

    #include <stdio.h>
void is_tool()
{
    printf("hello world");
}

#include <stdio.h>
void is_tool()
{
    printf("hello world");
}

int main()
{
    is_tool();
    return 0;
}

    可以看到is_tool()被重复定义了，于是编译器报错。

    将you.h换做以下代码后

    //you.h
#ifndef __you_h__
#define __you_h__
#include <stdio.h>
void is_tool()
{
    printf("hello world");
}
#endif // !__you_h__

    整体代码便是

    #ifndef __you_h__
#define __you_h__
#include <stdio.h>
void is_tool()
{
    printf("hello world");
}
#endif // !__you_h__

#ifndef __you_h__                   //由于上面代码中以定义宏__you_h__所以不编译#endif前代码
#define __you_h__
#include <stdio.h>
void is_tool()
{
    printf("hello world");
}
#endif // !__you_h__

int main()
{
    is_tool();
    return 0;
}

    即可成功编译运行

    3. 宏替换

    #define xxx yyy                           //在编译中代码里的xxx都会被yyy替代
eg：
        1.#define xxx int
        xxx y;              //int y

        2.#define xxx "hhh"
        printf(xxx);                       //printf("hhh");

        3.#define xxx {return 0;\               //宏中换行为\+回车
        }
        int main()
        xxx                          //int main(){return 0;}

#define 宏函数名(形参) 关于形参的式子
eg:
        1.#define fun(x) x*x
        fun(x+y)                //x+y*y+x宏仅会替代

        2.#define fun(x) No##x   //宏函数代替标识符时要想其形参被带入标识符要在其前面加##
        fun(1)                    //No1

        3.#define fun(x) printf(#x)   //#将形参变成字符串，而@#则将其变为字符
        fun(hhh)               //printf("hhh")

        4.#define fun(x) x*fun(x)       //产生循环则x*fun(x)里fun(x)不会被替换
        fun(1)                     //1*fun(1）

         5.#define fun(...) printf(__VA_ARGS__)  //__VA_ARGS__为...的预定义宏
         fun("hhh") ==  printf("hhh")   /*  ...为可变参数即该参数可没有，放参数列表
                                        末尾，xxx...即为...取名为xxx
                                        则printf(__VA_ARGS__)变为printf(xxx)*/

        6.#define fun(x,y...) printf(x,##y)  //##可消除因空参数产生的多余','
        fun("hhh")  //printf("hhh")

预定义宏：
 __DATE__   //字符串常量类型，表示当前所在源文件的编译日期，输出格式为Mmm dd yyyy（如May 27 2006）。
 __TIME__   //字符串常量类型，表示当前所在源文件的编译日期，输出格式为hh:mm:ss（如09:11:10）。
 __FILE__   //字符串常量类型，表示当前所在源文件名，且包含文件路径。
 __LINE__   //整数常量类型，表示当前所在源文件中的行号。
 __FUNCTION__  //字符串常量类型，表示当前所在函数名。

    宏的作用域是从定义开始到源文件结束。可用#undef 宏名 来取消宏，也可以去重新去定义这个宏。

    4. other

    #line number "filename"   //用于重新设定当前由__FILE__和__LINE__宏指定的源文件名字和行号。
#error  xxxx             //使预处理器发出一条错误消息，然后停止执行预处理。
#pragma                 //过于繁杂暂不解释

    register ：这个关键字请求编译器尽可能的将变量存在 CPU 内部寄存器中而不是通过内 存寻址访问以提高效率。注意是尽可能，不是绝对。因为寄存器的数量是有限的，定义过量的register 变量便有部分不成功。

    注意：register 变量必须是 能被 CPU 寄存器所接受的类型。意味着 register 变量必须是一个单个的值，并且其长度应小 于或等于整型的长度。 而且 register 变量可能不存放在内存中，所以不能用取址运算符“&” 来获取register 变量的地址。

    static ：static修饰的变量与函数其作用域均限在该文件(cpp)中，static修饰的变量在程序结束后才释放，且static修饰的全局变量在定义它之前的代码中要使用它则要加extern 进行声明。

    sizeof ：sizeof并不是函数，sizeof 在 计算变量所占空间大小时，括号可以省略，而计算类型大小时不能省略。

    if： 准确的float 变量与“零值”进行比较 应该写为if((fTestVal >= -EPSINON) && (fTestVal <= EPSINON))EPSINON 为定义好的精度 ,如果一个数落在[0.0-EPSINON,0.0+EPSINON] 这个闭区间内，我们认为在某个精度内它的值与零值相等；否则不相等。

    case：case 后面只能是整型或字符型的常量或常量表达式 。每个 case 语句的结尾绝对不要忘了加 break。老老实实的把每一种情况都用 case 语句来完成，而把真正的默认情况的处理交给 default 子句。

    const：const在c中是只读变量，注意它是变量，只是这个变量只能读取无法修改。

    const int *i     // int const *i 两者相等都指i所指的整型变量即*i不能改变
int * const i    //这个便指指针i不能发生变化

    volatile：

    extern：extern 可以置于变量或者函数前，以标示变量或者函数的定义在别的文件中。函数默认为extern可不加。extern int i=0; //i为另一文件的全局变量，这种写法是错误的会产生重复定义

    union：union中需注意大端序与小端序，小段序即高地址对高字节，低地址对低字节，大端序即相反。
